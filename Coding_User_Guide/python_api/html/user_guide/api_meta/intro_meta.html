Interacting with META#

General#

The basic idea behind the META API is to automate many repetitive and tedious procedures with the minimum user interaction and to perform specific tasks that are not covered by the standard META commands and MΕΤΑ session capabilities. Some of the tasks that can be performed within META API are the following:

  • Load models, scan and load resuts from solver output files.

  • Extract any type of information from an already loaded model (data from nodes, elements, boundary elements, materials, models, coordinate systems and groups etc).

  • Access and manipulate data from META entities like annotations, groups, cut planes and isofunctions.

  • Manipulate 2D plots: create and modify curves, run functions and access their attributes.

  • Create new results (deformation-scalar-vector) on nodes and elements or make calculations on existing ones.

  • Create and manipulate 3D and 2D windows.

  • Create custom GUI.

  • Use of session commands.

  • Running automatically (after launching META) a series of functions.

  • Communication with the interface (File Manager functions, Pick functions).

  • Profit from Python’s functionality to manipulate files and directories, read and write ASCII or binary files, use system commands etc.

These tasks are controlled through a series of MΕΤΑ specific functions, objects and methods.

To use a META specific function in Python, the relevant module has to be imported from META. The names of the modules are the available categories in the drop-down menu in tab Library in Functions List window. The functions are then called as methods of the imported module.

For example to print the ids and names of the all models

# PYTHON script
import meta
from meta import utils

def main():
    models = utils.get_models('all')
    for model in models:
        print('Id:   ' +str(model.id)+'   Name:   '+model.name)

Python objects are directly linked to META entities and not copies, as it is shown in the following three examples:

Example 1 - Access object after deletion of internal META entity

# PYTHON script
import meta
from meta import models

def main():
    window_name = 'MetaPost'
    filename = input.key
    deck = 'DYNA'
    r = models.LoadModel(window_name,filename,deck)
    if r:
        print('Model id: ',r.id)
    models.DeleteModel(r.id,window_name)
    print('Model id: ',r.id)

Copied objects can be used to maintain access to the initial object, after deletion of the internal META entity, as follow:

# PYTHON script
import meta
from meta import models

class CopiedModel():
    def __init__(self, obj):
        self.id = obj.id
        self.name = obj.name
        self.label = obj.label
        self.deck = obj.deck
        self.active = obj.active
        self.object = obj


def main():
    window_name = 'MetaPost'
    filename = input.key
    deck = 'DYNA'
    r = models.LoadModel(window_name,filename,deck)
    if r:
        r_copied = CopiedModel()
        print('Model id: ',r_copied.id)

    models.DeleteModel(r.id,window_name)
    print('Model id: ',r_copied.id)

Collect entities#

Introduction#

There is also a separate group of script functions for META entities. For example, there are Element functions to collect the elements, Node functions to collect the nodes, etc.

Additionally, there are the following generic functions models.CollectEntities to collect all the specified loaded entities:

Function

Description

meta.models.CollectEntities()

Collect all existing entities of the specified type in the current META session

meta.models.CollectModelEntities()

Collect all existing entities of the specified type in a mode

meta.windows.CollectWindowEntities()

Collect all existing entities of the specified type in a window

The types of entities that can be collected using these functions are: pages, models, parts, elements, nodes, plots, curves, annotations, cut planes e.t.c

Collect entities of the database#

For collecting all entities, the appropriate functions are the functions with just the type of the entities to be collected, e.g. Annotations(), Boundaries(), LoadsOfBoundary(), Elements(), Nodes(), Parts(), Models(), Groups(), Resultsets(), Connections(), etc. Please note that Groups refer to the entities Parts/Groups of Part Manager, Includes, Boundaries, Connections, Sets

In the case of model entities, the model id is given as argument.

To access the entities directly according to their id, the list with the model objects can be converted to a dictionary with the XxxListToDict() functions where the key is the entities’ id, e.g. AnnotationsListToDict(), ElementsListToDict(), etc.

# PYTHON script
import meta
from meta import parts

def main():

    model_id = 0
    all_parts = parts.Parts(model_id)
    for p in all_parts:
        print(p.id)
        print(p.name)

    dict_parts = parts.PartsListToDict(all_parts)
    for key, p in dict_parts.items():
        print(key)
        print(p)

    p = dict_parts[10] #Get directly part with id 10
    print(p.name)

To get the entities types of a model, the functions XxxTypeXxx() are available, e.g. ElementsTypes(), CurvesTypesNastran(), DeformationTypes(), ScalarTypes(), VectorTypes() etc.

# PYTHON script
import meta
from meta import elements

def main():
    model_id = 0
    all_types = elements.ElementsTypes(model_id)
    for element_type in all_types:
        print(element_type)

To retrieve deformation results types of a database the following can be used.

# PYTHON script
import meta
from meta import results

def main():
    filename = 'X:/project.metadb'
    deck = 'METADB'
    all_types = results.DeformationTypes(filename, deck)
    for one_type in all_types:
        total = len(one_type)
        deform_type = one_type[0] # Deformation type
        print(deform_type)
        for i in range(1, total):
            state = one_type[i]
            print(state)
            print(i) # Id of the state

For curves types of Dyna, Pamcrash and Radios files the functions plot2d.CurvesTypesDynaWithNames(), plot2d.CurvesTypesPamcrashWithNames() and plot2d.CurvesTypesRadiossWithNames() are available to get also their name.

Collect entities from other entities#

Using the Xxx.get_xxx() methods of the objects, it is possible to collect entities from other entities, e.g. parts.Part.get_elements(), elements.Element.get_nodes() , nodes.Node.get_elements() , groups.Group.get_parts(), etc.

# PYTHON script
import meta
from meta import models
from meta import groups

def main():
    m = models.Model(0)
    group = groups.Group('My_Group', m.id)
    group_parts = group.get_parts('all')
    for p in group_parts:
        print(p.id, p.type, p.subtype, p.visible, p.name, p.mat_id, p.shell_thick, p.model_id)

Collect entities according to their attributes#

In order to collect entities according to one of their attributes, Xxx.get_xxx() methods with named arguments can be used e.g. models.Model.get_nodes (‘with_name’, name = ‘<node name>’) , models.Model.get_nodes (‘with_name’, field10 = ‘<field name>’) , models.Model.get_elements (‘all’, element_id = <element id> ) , windows.Window.get_curves (‘byname’ , name = <curve name>) , models.Model.get_elements (‘all’, element_type = ‘<element type>’), models.Model.get_elements (‘failed’, resultset = <resultset>), models.Model.get_nodes (‘free’), parts.Part.get_elements (‘outer’), models.Model.get_nodes (‘follow’) etc.

# PYTHON script
import meta
from meta import models
from meta import constants

def main():
    m = models.Model(0)
    type =constants.PSHELL
    collected_parts = m.get_parts('all' , part_type  = type)
    for p in collected_parts:
       print(p.id,p.type,p.subtype,p.visible,p.name,p.mat_id,p.shell_thick,p.model_id)

For entities that need to be collected and which are neighbors of other entities, the argumnet ‘neighbour’ can be used e.g. models.Model.get_elements (‘neighbour’), materials.Material.get_materials (‘neighbour’), parts.Part.get_parts (‘neighbour’) etc. can be used. Neighboring entities are those which are directly attached to the specified entity or calculated by the solver.

For entities that need to be collected according to their nearest relative position to one given point, the argumnet ‘nearest’ can be used e.g. parts.Part.get_nodes (‘nearest, point_coordinates = point, distance_type = ‘xyz’), groups.Group.get_nodes (‘nearest, point_coordinates = point, distance_type = ‘xyz’), parts.Part.get_elements (‘nearest’, point_coordinates = point), groups.Group.get_elements (‘nearest’, point_coordinates = point ), materials.Material.get_nodes (‘nearest’, distance_type = ‘x’ , point_coordinates = point ), materials.Material.get_elements (‘nearest’, point_coordinates = point ) etc.

When entities need to be collected according to results (deformation, scalar, vector) or according to more advanced filters, then the functions AdvFiltersOnXxx() and XxxFromAdvFilters() can be used, e.g. AdvFiltersOnNodes(), AdvFiltersOnPoints(), ElementsFromAdvFilters(), etc.. The difference between AdvFiltersXxx() and XxxFromAdvFilters() functions is that in AdvFiltersXxx() the filters are given as a string argument to the functions, whereas in XxxFromAdvFilters() the execution of the script will stop and a window will open in order for the user to specify the advanced filters.

An easy way to build the string argument with the advanced filters for AdvFiltersXxx() is to apply an iFilter from GUI inside a META session. The string, that corresponds to the filters selected, will be shown as argument of the applied META command in the command line or in the current META_post.ses.

# PYTHON script
import meta
from meta import models
from meta import elements

def main():
    m = models.Model(0)
    adv_filters = list()
    adv_filters.append('add:Parts:id:==1:Keep All')
    adv_filters.append('intersect:Elements:centroidfuncmax::Max 3')
    all_resultsets = m.get_resultsets()
    result = all_resultsets[1]
    collected_elements = elements.AdvFiltersOnElements(model_id,adv_filters, result)
    for e in collected_elements:
        print(e.id)

For collecting the visible entities, the ‘visible’ specifier of the Xxx.get_xxx() methods should be used e.g. pages.Page.get_annotations (‘visible’), models.Model.get_elements (‘visible’), pages.Page.get_curves (‘visible’), etc.

The ‘visible’ specifier optionally combined with named arguments type, window e.t.c e.g. models.Model.get_elements (‘visible’ ,element_type = constants.SOLID)

Collect identified entities#

For getting only the identified entities of a model the respective specifier is ‘identified’ e.g. models.Model.get_elements (‘identified’), models.Model.get_nodes (‘identified’), models.Model.get_parts (‘identified’) etc.

Collect models information#

There is a series of functions to get information from the loaded models, like nodes.NumOfNodes, models.NumOfModels, parts.NumOfPartsByType(), etc.

Also information only for the active Models, Pages, Plots, Plot axes and Windows can be retrieved through the ‘active’ specifier of of the Xxx.get_xxx() methods e.g. utils.get_models (‘active’), utils.get_pages (‘active’), pages.Page.get_plots (‘active’), pages.Page.get_plot_axes (‘active’) and pages.Page.get_windows (‘active’).

# PYTHON script
import meta
from meta import utils

def main():
    act_models = utils.get_models('active')
    for r in act_models:
        print(r.id, r.name, r.label, r.deck, r.active)

Information about the currently loaded plot models (the plot files listed in the Read Results > Curves > Files List) that may exist can be retrieved through the function utils.get_plot_models().

# PYTHON script
import meta
from meta import utils

def main():
    plot_models = utils.get_plot_models()
    for pmod in plot_models:
        print(pmod.id, pmod.deck, pmod.filename)

Collect newly created entities#

For collecting all newly created entities of a specific type, the respective functions to use are CreateNewXxxStart(), ReportNewXxx() and CreateNewXxxEnd()

The syntax is, for example: CreateNewAnnotationsStart(), ReportNewAnnotations(), CreateNewAnnotationsEnd(), etc.

This can be very useful in order to get the number and ids of entities created through filters or session commands.

For example, for curves created through the Curve Function User Defined the function ReportNewXxx() will return the entity‟s objects without ending the recording of the created entities.

To collect newly created entities within a period, the functions to use are CollectNewXxxStart() and CollectNewXxxEnd()

# PYTHON script
import meta
from meta import utils
from meta import plot2d

def main():
    utils.MetaCommand('xyplot read lsdyna "Window1" "Z:/demo/nodout" Node 1 Magnitude_of_acceleration_(ma)')
    plot2d.CollectNewCurvesStart()
    utils.MetaCommand('xyplot curve function userdef "Displacement Difference" "c1.x" "c1.y-c2.y" "Window1"')
    new_curves = plot2d.CollectNewCurvesEnd()
    for c in new_curves:
        print(c.id,c.name)

Collect files and directories#

For collecting files (of specific formats) or directories, the commands of the python os module can be used. A simple example to get the files in a directory is the following:

# PYTHON script
import meta
import os

def main():
    path = 'Z:/demo'
    files = os.listdir(path)
    for f in files:
        print(f)

Select files or directories through the file manager#

META API interacts directly with the File Manager through the commands utils.SelectOpenDir(), utils.SelectSaveDir(), utils.SelectOpenFile(), utils.SelectSaveFile(), utils.SelectOpenFileIn(), utils.SelectSaveFileIn(). These functions open the File Manager and allow the selection or creation of files and directories. This is an elegant way to use file and directory paths in user scripts, since it enables the interactive definition of script parameters. The functions that deal with files return a list containing strings that represent the full path to the selected files, while those for directories return a string indicating the full path to the folder.

# PYTHON script
import meta
from meta import utils

def main():
    print('Select the file for reading')
    read_file = utils.SelectOpenFile(0, 'csv files (*.csv)')
    if not read_file:
        print('No file was selected')
    else:
        print('The file that was selected is:', read_file[0])

    print('Select the log file for writing the error messages')
    save_file = utils.SelectSaveFile()
    if not save_file:
        print('No file was selected')
    else:
        print('The file that was selected for writing errors is:', save_file[0])

    print('Select the directory where the META files are located')
    dir = utils.SelectOpenDir('');
    if not dir:
        print('No dir was selected')
    else:
        print('The selected directory is:', dir)

Create, Edit, Delete and Handle Entities#

Introduction#

The entities Annotations, Cut planes, Groups, Isofunctions, Windows, Models, Pages, Plots, Parts Curves, Images and Videos can be created edited and deleted through script functions. The entities nodes, elements, materials and boundaries CANNOT be created, edited or deleted in META in general, so this is also not possible through scripting. Coordinate systems can be created but cannot be deleted. Moreover, specific functions exist for other operations related to the entities, e.g. to show, hide, get results from them, etc.

Create entities#

For creating entities the respective functions are CreateXxx(), e.g. CreateEmptyAnnotation(), CreatePlane(), CreateGroupFromElements(), etc..

The entities can also be created by applying the respective session commands (see also chapter “META session commands for use in Scripting Language”). However, one advantage of creating the entities through script functions is that the function returns the objects of the created entities. For example, when creating a curve through scripting, the id of the curve is available.

# PYTHON script
import meta
from meta import annotations

def main():
    window_name = 'MetaPost'
    text = '10th Annotation'
    a = annotations.CreateEmptyAnnotation(window_name, text)
    if annotations.IsAnnotation(a):
        print(a.id)

Edit entities#

There is a series of script methods for editing directly entities and their settings, for example windows.Window.set_curves_settings(), windows.Window.set_annotations_settings(), annotations.Annotation.set_element(), planes.Plane.set_origin(), groups.Group.add_parts(), windows.Window.rotate_view(), etc. However, the full possibilities of editing entities are covered through the META session commands and can be achieved by using the MetaCommand syntax (see also chapter “META session commands for use in Scripting Language”).

# PYTHON script
import meta
from meta import planes
from meta import utils

def main():
    plane_name = 'plane_axis2'
    pl = planes.Plane(plane_name)
    #Change origin through script function
    orig = [ 102.3, 25.6, 30.8 ]
    ok = pl.set_origin(orig)
    #Change section width through session command
    utils.MetaCommand('plane options width 3 "'+plane_name+'"')

Delete entities#

For deleting entities the respective Xxx.delete() is used, e.g. annotations.Annotation.delete(), plot2d.Curve.delete(), models.Model.delete(), etc.

The entities can also be deleted by applying the respective session commands.

Show / Hide entities#

For showing / hiding entities the respective method is Xxx.show() and Xxx.hide(), e.g. annotations.Annotation.show(), plot2d.Curve.hide() etc.

When more than one 3d model entities or curves need to be shown / hidden, it is much faster to use the methods Xxx.show_xxx() and Xxx.hide_xxx(), e.g. models.Model.show_elements(),

models.Model.hide_parts(), windows.Window.show_curves(), etc. In this case the entities must be given as list. The entities can also be shown / hidden by applying the respective session commands.

Identify entities#

For identifying 3d model entities the respective method is Xxx.identify , e.g. nodes.Node.identify(), elements.Element.identify(), materials.Material.identify(), etc.

When more than one 3d model entities need to be identified, it is much faster to use the methods Xxx.identify_xxx(), e.g. models.Model.identify_elements(), models.Model.identify_parts(), etc. In this case the entities must be given as list.

The entities can also be identified by applying the respective session commands.

Get /handle material properties#

To get the properties of material the method materials.Material.get_property() can be used. To set these attributes the script method materials.Material.set_property() can be sused.

Get results/attributes of entities#

First of all, to get the states for which the resultsets are needed, the methods that can be used are models.Model.get_current_resultset(), models.Model.get_resultsets(), models.Model.get_filtered_resultsets(), results.Result.get_resultsets().

The ‘generated’ specifier can be used to results.Result.get_resultsets() method get the results that generated from a specifc resultset, e.g. results.Result.get_resultsets(‘generated’)

If more than one labels exist for the resultset, the specifiers ‘deformations’, ‘scalar’, ‘vector’, ‘function’ can be defined to get directly all the label results of the state. e.g. results.Result.get_resultsets(‘deformations’), results.Result.get_resultsets(‘scalar’) e.t.c.

Alternatively, the methods results.Result.get_string_deformation_labels(), results.Result.get_string_scalar_labels(), results.Result.get_string_vector_labels() can be used to get the available labels and then the method results.Result.get_resultsets() can be used to get the specific label result of the state.

After the needed resultset is found, the functions to get the loaded results depend on the entity from which the result which is needed. So, in the Nodes functions group there are methods to get results on the nodes, in the Elements methods group there is method to get results on elements, in the Parts methods group method to get all results from a part, in the Materials methods, method to get all results from a material, in the Groups methods group method to get all results from a group and in the Models methods group there are methods to get all results from a model. Moreover, there is a method for each specific result needed. For nodes, to get the deformation results, there is the function nodes.Node.get_deformations(), to get the coordinates nodes.Node.get_coordinates(), to get scalar results nodes.Node.get_nodal_scalar() and to get vector results nodes.Node.get_nodal_vector().

Similarly for results on elements, there are methods elements.Element.get_deformations(), elements.Element.get_centroid_scalar(), elements.Element.get_corner_scalar(), elements.Element.get_nodal_scalar() and more.

For parts / materials / models there are methods that return the results on all the nodes or elements, e.g. parts.Part.get_centroid_vector()

Also ‘max’, ‘min’ specifier can be used to get the max/min value of results e.g. models.Model.get_nodal_scalar (<resultset>,’min’), elements.Element.get_deformations (<resultset>,’max’)

For CELAS spring connection elements the methods elements.Element.get_stiffness(), elements.Element.get_damping() and elements.Element.get_mass() can be used to get information about the stiffness, damping and mass respectively.

To get and set the number of steps of an axis the methods plot2d.PlotAxis.get_step() and plot2d.PlotAxis.set_steps() respectively can be used.

To get, calculate, and set the material tension, compression, shear, x_tension, y_tension, x_compression, y_compression, shear and f12 limits of composites materials, the methods materials.Material.get_material_limit() and parts.Part.get_material_limit() or materials.Material.set_material_limit() and parts.Part.set_material_limit() can be used respectively.

# PYTHON script
import meta
from meta import results
from meta import models
from meta import nodes

def main():
    m = models.Model(0)
    result = m.get_current_resultset()
    result = results.CurrentResultset(model_id)
    node_id = 10
    n = nodes.Node(id=node_id, model_id=m.id)
    deform = n.get_deformations(result)
    if results.IsDeformation(deform):
        print(deform.x, deform.y, deform.z, deform.total, deform.node_id)

import meta

        print(max_nodal.part_id) #Id of the part or -1 if no part exists

Get measurements from entities#

The functions to get measurements depend on the entity from which the measurement is needed. So, in the Nodes methods group there are methods to get measurements from the nodes, in the Elements methods group there are methods to get measurements from the elements, in the Parts methods group methods to get measurements from a part and in the Groups methods group methods to get measurements from a group.

Moreover, there is a different function for each specific measurement needed. To get the distance between a node and another node there is the method nodes.Node.get_distance_from_node(), to get the distance between a node and a part the function is nodes.Node.get_distance_from_part(), etc. To get the angle formed by nodes the function is AngleOfNodes(). To get the distance between a part and a group the method is parts.Part.get_distance_from_group().

# PYTHON script
import meta
from meta import parts
from meta import models
from meta import constants
from meta import groups

def main():
    m = models.Model(0)
    part = parts.Part(id=2, type=constants.PSHELL, model_id=m.id)
    res = m.get_current_resultset()
    group = groups.Group('My_Group', m.id)
    distance = part.get_distance_from_group(res, group, res)
    if len(distance):
        dist_x = distance[0] # Distance in direction X
        dist_y = distance[1] # Distance in direction Y
        dist_z = distance[2] # Distance in direction Z
        dist_total = distance[3] # Total distance
        print(dist_x, dist_y, dist_z, dist_total)

Get connected and neighbour elements/nodes#

In cases that the connected elements or nodes are needed, the methods models.Model.get_elements(), elements.Element.get_nodes() and nodes.Node.get_elements() should be used. In case of mesh independent spotweld connections, where there is no node connectivity, the specifier ‘neighbour’ can still be used as long as the connectivity is supported in META. e.g. models.Model.get_elements(‘neighbour’)

The above script functions should be preferred from the visibility session commands, e.g. “add connected”, as the first are much faster.

# PYTHON script
import meta
from meta import elements
from meta import models
def main():
    m = models.Model(0)
    neighbour_elements = m.get_elements('neighbour', element_type = 3 , element_id = 1,neighbour_type = 3)
    for e in neighbour_elements:
        print(e.id)

In cases that the connected elements, nodes, parts etc. to connections are needed, the methods connections.Connection.get_xxx() should be used.

# PYTHON script
import meta
from meta import connections
from meta import models

def main():
    m = models.Model(0)
    connection_id = 100022
    con = connections.Connection(connection_id, m.id)
    connected_elements = con.get_elements('all')
    for e in connected_elements:
        print(e.id)

Get spreadsheet entities/data#

There are functions to get data stored in a spreadsheet. To get a cells data the method spreadsheet.Sheet.get_cell() can be used.

Also the entire row‟s or column‟s data can be stored in a list through the methods spreadsheet.Sheet.get_row() and spreadsheet.Sheet.get_column(). Returned lists stops at the last non-empty cell. If all cells are empty, an empty list is returned

To get cells data of an entire cells area the method spreadsheet.Sheet.get_range() and function SpreadsheetCellsByLabel(sheet_name, cell_label) can be used. Cells data are stored in a list of same size as the defined spreadsheet area.

Also in order to identify the spreadsheet cells area that involves data the function spreadsheet.BoundingAreaOfSpreadsheet(sheet_name) can be used. A list that determines the bounding box of non-empty cells (list[0][0]: top-row, list[0][1]: left-column, list[1][0]: bottom-row, list[1][1]: right-column) will be returned.

Get report entities/data#

For getting entities of a report, report.Xxx.get_xxx() methods can be used e.g. report.Report.get_slides(), report.Slide.get_shape(), report.Textbox.get_text(), report.ExcelTable.get_size(), report.Table.get_cell() e.t.c.

There are also report.Xxx.set_xxx() methods to set the value on the relative entities e.g. report.Report.set_slide_size(), report.Slide.set_name(), report.Textbox.set_text(), report.ExcelTable.set_visibility(), report.Table.set_size() e.t.c.

In order to create the a report composer window there is the function report.CreateReport() and there are report.Xxx.add_xxx() methods to add new entities on the report e.g. report.Report.add_slide(), report.Slide.add_textbox(), report.Slide.add_excel_table(), report.Slide.add_table() e.t.c

User Toolbar Development Kit(TDK) is also available for a robust User Toolbar development. This class contains base functionality, which is common for all Python-bases User Toolbars. The kit’s Python module is provided as open source code for better understanding. Load Geometry and Solver Results, Create User-Defined Results#

Load geometry and field / history results#

For loading geometry the respective functions are models.LoadModel() for solver geometry data files and models.LoadProjectModel() for geometries from META databases and META projects.

For loading field results the functions are results.LoadDeformations(), results.LoadModalDeformations(), results.LoadScalar() and results.LoadVector() for solver results data files and results.LoadProjectDeformations(), results.LoadProjectScalar() and results.LoadProjectVector() for results from META databases and META projects.

The argument “data” which is needed for the functions is the same as the argument used in the equivalent META session commands. The best way to retrieve the correct syntax is to load the result manually from the META interface and view the session command passed to the current META_post.ses file.

In order to append the loaded geometry/results the functions results.LoadAppendDeformations(), results.LoadAppendProjectDeformations(), results.LoadAppendScalar(), results.LoadAppendScalarGetCoordSystems(), results.LoadAppendProjectScalar(), results.LoadAppendVector(), results.LoadAppendVectorGetCoordSystems(), results.LoadAppendProjectVector() are available.

For loading curves from history results files the respective functions are plot2d.LoadCurvesXxx(), e.g. plot2d.LoadCurvesNastran(), plot2d.LoadCurvesDyna().

The arguments which are needed for the functions, e.g. entities, variables, etc, are the same as the arguments used in the META session commands. The best way to retrieve the correct syntax is to load the result manually from the META interface and view the session command passed to the current META_post.ses file. The only difference is that the results are passed as matrices to the script functions. The string expressions can be easily changed to matrices and vice versa through the functions utils.RangeToList() and utils.ListToRange().

Especially for loading curves from RADIOSS history results files, the function plot2d.LoadCurvesRadioss() needs the time history id as argument. In case the time history id is not known, the function plot2d.GetRadiossTimeHistoryId() can be used to find the time history id for a specific history result.

The entities can also be loaded by applying the respective session commands (see also chapter “META session commands for use in Scripting Language”). However, one advantage of loading the entities through script functions is that the function returns the objects of the created entities. For example, when loading a model through the script function, the id of the model is available.

# PYTHON script
import meta
from meta import models
from meta import results

def main():
    window_name = 'MetaPost'
    filename = '/home/demo/example.op2'
    deck = 'NASTRAN'
    r = models.LoadModel(window_name, filename, deck)
    if models.IsValidModel(r):
        model_id = r.id
        states = '1-3'
        data = 'Displacements,Translational'
        new_resultsets = results.LoadDeformations(model_id, filename, deck, states, data)
        data = 'Stresses,VonMises,MaxofTopBottom'
        new_resultsets = results.LoadScalar(model_id, filename, deck, states, data)
        for res in new_resultsets:
            print(res.name,res.nodal_data_name,res.function_data_name,res.state)

Create user-defined field results#

Apart from loading solver results, it is possible to create user defined field results using the results from calculations realized inside the script.

First of all a new empty state for field results can be created with the function results.CreateResultSet(). To add a new label to the resultset and set the results to the nodes or the elements, the functions to be used are results.AddDeformationOnAllNodes(), results.AddNodalScalarOnAllNodes() and results.AddNodalVectorOnAllNodes().

# PYTHON script
import meta
from meta import results

def main():
    model_id = 0
    all_resultsets = results.Resultsets(model_id)
    state_id = len(all_resultsets)
    name = 'Script added resultset'
    cycle = 0
    new_resultsets = results.CreateResultset(model_id, state_id, name, cycle)
    for res in new_resultsets:
        print(res.cycle, res.model_id)
        print(res.name, res.nodal_data_name, res.function_data_name, res.vector_data_name)
        print(res.filename, res.subcase, res.state)
        print(res.step, res.frequency, res.time)
        print(res.mode, res.eigenvalue, res.imaginary_eigenvalue)
        print(res.loadstep, res.generate_sequence)
        print(res.nodal_data_label, res.function_data_label, res.vector_data_label)

For more complicated procedures:

To reset all the values of a resultset and add values to the nodes or the elements, the functions to be used are StartAddingXxx() and EndAddingXxx(), e.g. StartAddingDeformations(), EndAddindDeformations().

# PYTHON script
import meta
from meta import results
from meta import nodes

def main():
    m = models.Model(0)
    all_resultsets = results.Resultsets(model_id)
    resultset = all_resultsets[1]
    new_nodal_data_name = 'Deformations Data'

    node_ids = []
    xdeform  = []
    ydeform  = []
    zdeform  = []

    visnodes = m.get_nodes('visible')
    for n in visnodes:
        node_ids.append(n.id)
        xdeform.append(0.025)
        ydeform.append(-0.91)
        zdeform.append(-1.398)

    meta.results.StartAddingDeformations(resultset, new_nodal_data_name)
    meta.results.AddDeformationOnSomeNodes(resultset, node_ids, xdeform, ydeform, zdeform)
    meta.results.EndAddingDeformations(resultset)

To change the values of a resultset the functions to be used are results.StartChangingXxx() and results.EndAddingXxx(), e.g. results.StartChangingCentroidVector(), results.EndAddingCentroidVector().

To append a new label to a resultset and set values to the nodes or the elements for this label, the functions to be used are results.StartAppendingXxx() and results.EndAddingXxx(), e.g. results.StartAppendingCentroidScalar(), results.EndAddingCentroidScalar().

Between the StartXxx() and EndXxx() functions the values can be assigned to the nodes with the functions results.AddCentroidScalarOnElement(), results.AddCentroidScalarOnSomeElements(), results.AddCentroidVectorOnElement(), results.AddCentroidVectorOnSomeElements(), results.AddCornerScalarOnElement(), results.AddCornerScalarOnSomeElements(), results.AddDeformationOnNode() and results.AddDeformationOnSomeNodes(). When values are to be to more than one elements / nodes, functions results.XxxOnSomeElements() and results.XxxOnSomeNodes() should be preferred, as they are much faster.

Note

The results.EndAddingXxx() function must be called at the end, or else the values will not be assigned to the resultset.

Useful notes about META script functions#

Speed up the execution of scripts#

When executing scripts some points should be taken into consideration in order to achieve minimum execution times.

  • When the script includes functions that change the display of the META windows, each redraw will result to time consumption. To speed up the execution of the script, the redraws can be disabled and enabled only at the end of the script, in order to display directly the final state.

This can be achieved through the lines:

MetaCommand('options session controldraw disable')
MetaCommand('options session controldraw enable')

  • When a function needs to be applied on more than one entities, it is always faster to use the function that applies directly on all the needed entities XxxOnSomeXxx, e.g. AddElementScalarOnSomeElements(), etc, instead of applying the single function for each entity.

  • When all the results from entitiy are needed, it is faster to use the method that gets all the results directly, e.g. models.Model.get_deformations(), parts.Part.get_centroid_scalar(), etc, instead of getting the result for each element / node separately.

  • In cases the connected elements or nodes are needed the functions models.Model.get_elements(‘neighbour’), elements.Element.get_nodes() and nodes.Node.get_elements() should be preferred from the visibility session commands, e.g. “add connected”, as the first are much faster.

User defined buttons and custom GUI#

General#

META API enables the creation of user defined buttons and fully customized graphical interfaces. The buttons that can be created are similar to the buttons of the GUI and are used to invoke user functions. For the management of specific tasks that must be controlled through a number of definitions and actions, it is very useful to create a custom GUI. There are two libraries of functions to create custom user interface.

User Toolbars

The first supports only the basic widgets that a GUI can hold, like checkbuttons, radio buttons, menu buttons, lists, tables.These GUI items are generated without the need to type any GUI generating code, which is more difficult and time consuming to do than creating a Toolbar.

This simplicity comes at the cost of flexibility and customizability, as Toolbars offer only a predefined set of capabilitities, which are enough for the majority of the cases where a special GUI is needed. All Toolbars are available under the User Toolbars menu of META. Toolbar Designer (User Toolbars>Toolbar Designer) tool is also available for the interactive creation / modification of toolbars.

GUITK Functionality

While the second library, GUITK, is more extensive and except from the main widgets mentioned before it also supports hundreds of more sophisticated tools. These tools allow the creation of any complex interface that may also contain tab widgets, popup menus, spin boxes, group of buttons, tooltips and so on. For more information check the guitk Module.
