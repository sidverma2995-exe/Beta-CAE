META Remote Control#

Introduction to Remote Control#

META offers the capability to be executed remotely, so that a user can remotely “stream” session commands to it.

This procedure can be accomplished by subclassing the MetaProcess class of the MetaProcessModule.py, which can be found in the ANSA installation directory, under the path: /scripts/RemoteControl/meta/MetaProcessModule.py.

Subclassing the MetaProcess class#

In order to subclass the MetaProcess class, the user needs to create a class of his own, which will inherit from the MetaProcess class.

An example of such a class can be seen below.

class MetaRemoteControl(MetaProcess):
    def __init__(self, meta_command=None, run_in_batch=True, other_running_options=list()):
        super().__init__(meta_command, run_in_batch, other_running_options)
        super()._establish_connection()

    def execute_script(self):
        # Read and execute a python file
        cur_path = os.path.dirname(os.path.realpath(__file__))
        script_path = os.path.join(cur_path, 'test_script_meta.py')
        self._send_cmd('read session ' + script_path)

    def clear_meta(self):
        self._clear_meta()

    def terminate_process(self):
        self._terminate_process()

Establishing the Connection#

The connection with META is established by executing the __init__() and the _establish_connection() methods of the parent class. Typically, this should happen in the __init__() method, through the super() function.

Specifically, the super().__init__() command, is responsible for opening a META Process and it has 3 arguments.

  • meta_command defines the path to the META executable file. If it’s not defined, the MetaProcessModule will try to find it by itself, through ANSA’s installation directory.

  • run_in_batch is a boolean variable declaring whether META will be opened with or without GUI.

  • other_running_options is a list of further execution options.

The command super().establish_connection() is responsible for attaching to the previously opened META Process.

def __init__(self, meta_command=None, run_in_batch=True, other_running_options=list()):
    super().__init__(meta_command, run_in_batch, other_running_options)
    super()._establish_connection()

Executing Commands#

After establishing the connection with META, the user is now able to execute session commands on it.

This can happen by executing the _send_cmd() method of the parent class and simply providing the string command to be executed.

In our example, this is taking place inside the execute_script() method, where a ‘read session’ command is executed.

def execute_script(self):
    # Read and execute a python file
    cur_path = os.path.dirname(os.path.realpath(__file__))
    script_path = os.path.join(cur_path, 'test_script_meta.py')
    self._send_cmd('read session ' + script_path)

Terminating the Connection#

After executing the desired commands, the user should terminate the META Process, through the method _terminate_process().

def terminate_process(self):
    self._terminate_process()

Usage Example#

The following example demonstrates a simple usage scenario of the class that we created.

First establishing the connection, then executing a command in META and finally terminating the connection.

def main():
    # Establish the connection
    meta_process = MetaRemoteControl()

    # Execute the desired commands
    meta_process.execute_script()

    # Clear and Terminate META
    meta_process.clear_meta()
    meta_process.terminate_process()

Note

All the aforementioned examples’ code can be found in the ANSA installation directory, under the path: /scripts/RemoteControl/meta_examples/
