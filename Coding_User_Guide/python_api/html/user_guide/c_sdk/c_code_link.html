Interfacing Python with C Code#

Interfacing Python with C code is important in scientific and engineering computing. The C programming language is very fast and efficient when it comes to intensive computations. Thus being able to extend the Python functionality for such purpose is a powerful feature. There are two ways of achieving this.

  • The first approach is by using Python ctypes library. ctypes is glue code between Python and C shared libraries (linux) or DLLs (windows).

  • The second appropach is by using Python C API. This API gives the programmer a direct access to the Python interpreter and it gives the ability to develop extension modules. This is the approach that was used in order or develop the ANSA and META Python APIs.

The tutorials that will be introduced in this section will cover both cases. cmake and setup.py files are provided so that the programmer does not have to deal with figuring out the compilation processes. README files describe the setup process in detail. The package with all the files can be downloaded from here Python module samples.

Note

Please refer to the previous section for a list of compilers that guarantee binary compatibily

Warning

The version of Python embedded in ANSA and META is v3.11.2. Thus a local installation of this version is needed in order for the C modules to compile v3.11.2 compatible binaries

Binding C Code with ctypes#

ctypes is a glue code between python and C. There is no need to do any type of code intervention in C code. ctypes are also used to re-use legacy C code. The only requirement is that the code is compiled as a Shared library (.so) or DLL (for windows)

The example below will demostrate a C program that calculates the Fibonacci number for a given sequence index. The code is shown below:

#include <stdio.h>

int Fibonacci(int n)
{
    int i = 0;
    int first = 0, second = 1, next = 0;

    for (i = 0; i < n; i++) {
        if (i <= 1) {
            next = i;
        } else {
            next = first + second;
            first = second;
            second = next;
        }
    }
    return next;
}

This program can be made to be called by a Python program. The Python code can be seen below:

import ctypes
import os


def main():
    d = os.path.dirname(os.path.realpath(__file__))
    library_path = os.path.join(d, 'build', 'libsample_c_library.so')
    lib = ctypes.cdll.LoadLibrary(library_path)

    restype = ctypes.c_int
    argtypes = [ ctypes.c_int ]
    fibonacci_prototype = ctypes.CFUNCTYPE(restype, *argtypes)
    fibonacci_function = fibonacci_prototype(lib.Fibonacci)

    term = 10
    nf = fibonacci_function(term)
    print('The {}th Fibonacci number is {}'.format(term, nf))

if __name__ == '__main__':
    main()

Let us go line by line for a more detailed look of the code.

import ctypes

This line imports the ctypes library.

d = os.path.dirname(os.path.realpath(__file__))
library_path = os.path.join(d, 'build', 'libsample_c_library.so')
lib = ctypes.cdll.LoadLibrary(library_path)

The first two lines construct the path of the location of the C shared library. In the third line the C library is loaded. The LoadLibrary method creates an instance of the CDLL Class. The data of this instance is the loaded shared library.

restype = ctypes.c_int
argtypes = [ ctypes.c_int ]
fibonacci_prototype = ctypes.CFUNCTYPE(restype, *argtypes)
fibonacci_function = fibonacci_prototype(lib.Fibonacci)

The CFUNCTYPE function constructs a C function prototype for a given callback function. The arguments are the return type of the lib.Fibonacci function and the argument of the lib.Fibonacci given in a list. The function will take an integer as an argument and the return values type will also be an integer. The ctypes.c_int data type is C’s signed int type. In the last line the lib.Fibonacci is passed to the C prototype and a python

term = 10
nf = fibonacci_function(term)

The fibonacci_function can be used like any other python function

Extending Python with the C API#

The Python C API allows the programmer to write extensions modules. In this example an extension module called Porch will be created. This module will have three methods (functions). The creation of sunflower seeds, which follow a fibonacci sequence pattern, and the creation of rose petals by solving parametric equations. Below the C and the python codes are presented. Additionaly the plants_colors function returns the color of a given flower. The new concepts of the C API are explained in detail.

#include <Python.h>
#include <stdio.h>
#include <math.h>
#include <string.h>

static PyObject *sunflower_seeds(PyObject *self, PyObject *args)
{
    int first = 0, second = 1, next = 0;
    int n, i;

    if (!PyArg_ParseTuple(args, "i:porch.sunflower_seeds", &n)) return NULL;

    if (n < 0) {
        PyErr_SetString(PyExc_ValueError, "porch.sunflower_seeds: Spiral value should be positive or zero!");
        return NULL;
    }

    for (i = 0; i < n; i++) {
        if (i <= 1) {
            next = i;
        } else {
            next = first + second;
            first = second;
            second = next;
        }
    }
    return PyLong_FromLong(next);
} /* sunflower_seeds */


static PyObject *rose_function(PyObject *self, PyObject *args, PyObject *kwds)
{
    PyObject    *py_dict = NULL, *py_x_coor = NULL, *py_y_coor = NULL;
    int         t_param = 0, k_factor = 0;
    double      x_coor, y_coor;
    const char  *kwlist[] = { "k_factor", "t_parameter", NULL };

    if (!PyArg_ParseTupleAndKeywords(args, kwds, "ii:porch.rose_function", (char **)kwlist, &k_factor, &t_param)) {
        return NULL;
    }

    x_coor = cos(t_param * k_factor) * cos(t_param);
    y_coor = cos(t_param * k_factor) * sin(t_param);

    py_dict = PyDict_New();

    py_x_coor = PyFloat_FromDouble(x_coor);
    PyDict_SetItemString(py_dict, "x_coordinate", py_x_coor);
    Py_DECREF(py_x_coor);

    py_y_coor = PyFloat_FromDouble(y_coor);
    PyDict_SetItemString(py_dict, "y_coordinate", py_y_coor);
    Py_DECREF(py_y_coor);

    return py_dict;
} /* rose_function */


static PyObject *plants_colors(PyObject *self, PyObject *args)
{
    char    *flower_name = NULL;
    char    msg[32], out_msg[256];

    if (!PyArg_ParseTuple(args, "s:porch.plants_colors", &flower_name)) {
        return NULL;
    }

    if (!strcmp(flower_name, "rose")) {
        sprintf(msg, "red");
    } else if (!strcmp(flower_name, "sunflower")) {
        sprintf(msg, "yellow");
    } else if (!strcmp(flower_name, "olive tree")) {
        sprintf(msg, "green");
    } else if (!strcmp(flower_name, "daisy")) {
        sprintf(msg, "white");
    } else {
        PyErr_SetString(PyExc_ValueError, "porch.plants_colors: invalid flower_name");
        return NULL;
    }

    sprintf(out_msg, "My porch's %ss are %s", flower_name, msg);

    return PyBytes_FromString(out_msg);
} /* plants_colors */


static PyMethodDef PorchMethods[] = {
    {
        "sunflower_seeds", sunflower_seeds,
        METH_VARARGS,
        "Calculate the fibonacci number of the seed heads in the main sunflower spiral in the n-th ring."
    },
    {
        "rose_function", (PyCFunction)rose_function,
        METH_KEYWORDS | METH_VARARGS,
        "Calculate the rose function's coordinates, with given parameter ( t ) and factor ( k )."
    },
    {
        "plants_colors", plants_colors,
        METH_VARARGS,
        "Return my porch's plant colors. Input is a plant name (rose, sunflower, olive tree, daisy)"
    },
    { NULL }
};


static struct PyModuleDef porch = {
    PyModuleDef_HEAD_INIT,
    "porch",
    NULL, //Documentation
    -1, // Default value in order to keep state in global variables.
    PorchMethods
};


PyMODINIT_FUNC PyInit_porch(void)
{
    PyObject *m;

    m = PyModule_Create(&porch);

    return m;
} /* PyInit_fibonacci */

#include <Python.h>

This includes the python API header into the program. You may need to install the python-dev package in order to get the development files.

static PyObject *sunflower_seeds(PyObject *self, PyObject *args)

This C function defines a Python Object sunflower_seeds.

if (!PyArg_ParseTuple(args, "i:porch.sunflower_seeds", &n)) return NULL;

In this line the function arguments are parsed for a single integer number. If no integer is given the function returns NULL and an exception is raised.

PyErr_SetString(PyExc_ValueError, "porch.sunflower_seeds: Spiral value should be positive or zero!");
return NULL;

If a negative or a zero value is given a ValueError exception is raised.

return PyLong_FromLong(next);

The C long integer is converted to a Python integer.

static PyObject *rose_function(PyObject *self, PyObject *args, PyObject *kwds)

This C function defines a Python Object rose_function. Note that this function has an extra argument PyObject *kwds. This is in order to define named arguments(keywords).

char *kwlist[] = {"k_factor", "t_parameter", NULL};

This array lists the names of the named arguments. It must always be NULL terminated.

if (!PyArg_ParseTupleAndKeywords(args, kwds, "ii:porch.rose_function", kwlist, &k_factor, &t_param)) {
    return NULL;
}

The function parses the arguments in order to find two integers. These two integers will be assigned to k_factor and t_param either by position or by name.

py_dict = PyDict_New();

py_x_coor = PyFloat_FromDouble(x_coor);
PyDict_SetItemString(py_dict, "x_coordinate", py_x_coor);
Py_DECREF(py_x_coor);

py_y_coor = PyFloat_FromDouble(y_coor);
PyDict_SetItemString(py_dict, "y_coordinate", py_y_coor);
Py_DECREF(py_y_coor);

A python dictionary is created and python floats are assigned as values.

static PyObject *plants_colors(PyObject *self, PyObject *args)

This C function defines a Python Object plants_colors.

if (!PyArg_ParseTuple(args, "s:porch.plants_colors", &flower_name)) {
    return NULL;
}

Parses the arguments to find a string of the flower name.

static PyMethodDef PorchMethods[]

This array defines the python methods

static struct PyModuleDef porch = {
    PyModuleDef_HEAD_INIT,
    "porch",
    NULL, //Documentation
    -1, // Default value in order to keep state in global variables.
    PorchMethods
};

This struct defines the python module.

PyMODINIT_FUNC PyInit_porch(void)

This C function creates the python module.

The python code that will that will use the created modules is shown below. The created library after compilation is imported and the methods can be used with standard python syntax.

import sys
import os
dir = os.path.dirname(os.path.dirname(os.path.realpath(__file__)))
path = os.path.join(dir, 'build', 'lib.linux-x86_64-3.11')
if path not in sys.path: sys.path.append(path)
import porch


def test():
    seed_num = porch.sunflower_seeds(5)
    print(type(seed_num), seed_num )
    coords = porch.rose_function(5, 1)
    print(coords)

    print(porch.plants_colors('rose'))


if __name__ == '__main__':
    test()
