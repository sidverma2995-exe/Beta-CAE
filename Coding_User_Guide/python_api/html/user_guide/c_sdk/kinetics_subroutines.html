C SDK for Kinetics Subroutines#

The C SDK can be used to customize the Kinetics Solver to model non-standard phenomena.

The Software Development Kit#

Location#

The provided SDK (Software Development Kit) is called Kinetics Subroutines. It can be downloaded from here Kinetics Subroutines

Installation#

The package needs to be unzip. The contents of the package are:

Directory/File

Description

CMakeLists.txt
The cmake file. Depending on your computer’s OS you have to use the
correct target_link_library and set the correct path.

gse_deriv.c
The user’s subroutine Gse_deriv for the GSE example. It must follow a
specific prototype that is defined in kinetics_c_utils.h

gse_output.c
The user’s subroutine Gse_output for the GSE example. It must follow a
specific prototype that is defined in kinetics_c_utils.h

kinetics_c_utils.h
The header file with our API functions. It must be included in the
source file like in the examples gse_deriv.c, gse_output.c

beta-kinetics.lib
Library to be used in CMakeLists.txt in target_link_library path
(Windows only)

spring_mass_system_as_gse.ansa

The model you can use in order to test the created library.

Adding your own subroutine#

In the kinetics_subroutines directory , there are two files , named gse_deriv.c and gse_output.c, which should contain the C implementation of user-written subroutines Gse_deriv and Gse_output, respectively .These files currently contain a Gse_deriv and a Gse_output implementation , as an example case. Note that if the user wants to use a different file than the given gse_deriv.c or gse_output.c, the CMakeLists.txt needs to be edited as follows:

set(SOURCES gse_deriv.c
            gse_output.c)

The C functions must follow the prototypes below:

void Gse_deriv(const struct sAnsaGSE* gse, double TIME, int DFLAG, int IFLAG, int NS, double* XDot);

void Gse_output(const struct sAnsaGSE* gse, double TIME, int DFLAG, int IFLAG, int NO, double* Y);

Now compile the project and load the .so or .dll in ANSA.

Setting up the compilation#

Setting up the make file#

  1. In both Linux and Windows users have to edit CMakeLists.txt. Only in case of Windows, modify the target_link_libraries path depending on the location of the shared_v21.1.0/win64/lib64/ directory in your local installation, as shown below:

target_link_libraries(gse_user_library
    #C:/path/to/beta-kinetics.lib # Windows
)

For Linux users#

Run the following commands:

cd template/build

cmake ..

make

For Windows users#

  1. Go to the template/build folder

  2. Run

cmake -G "Visual Studio 15 2017 Win64" ..

  1. Double click on the solution (.sln) file to open it with Visual Studio

  2. Build the project

Both Linux and Windows#

The compilation will create the template/build/libgse_user_library.so (Linux) or template/build/Debug/gse_user_library.dll (Windows) library.

Note

A single so or dll needs to be provided for all C user-written subroutines

Note

The above compilation instructions need to adhere to the Binary Compatibility instructions

Adding Library in ANSA#

In order to load the compiled library of user defined subroutines follow the steps:

  1. Copy the Library created in the previous step (so or dll) to the .BETA or in the ANSA_HOME directory.

  2. Go to Deck: Kinetics > Simulator > Options > Tab: General > Subroutine Settings

  3. Load the created shared library and press OK.

  4. Go to KIN_EQUATION of type GSE

  5. Select SCRIPT_FUNCTION_TYPE = C_FUNC

  6. Add USER_FUNC_DERIV = Gse_deriv (function name defined in gse_deriv.c), USER_FUNC_DERIV_ARGS = 1.0 (function arguments)

  7. Add USER_FUNC_OUTPUT = Gse_output (function name defined in gse_output.c), USER_FUNC_OUTPUT_ARGS = 1.0 (function arguments)

  8. Press OK

  9. Go to Simulator > Run

  10. User functions should be taken into account during simulation.

void Gse_deriv(const struct sAnsaGSE *gse, double TIME, int DFLAG, int IFLAG, int NS, double *XDot)#

This function defines the continuous function fc()

Parameters:

  • gse – an object that contains information for gse element

  • TIME – the current simulation time

  • DFLAG – should be ignored

  • IFLAG – indicates why the routine is being called

  • NS – the number of continuous states in gse

  • XDot – an array that contains the return values (the derivatives of the state xc)

void Gse_output(const struct sAnsaGSE *gse, double TIME, int DFLAG, int IFLAG, int NO, double *Y)#

This function defines the continuous function g()

Parameters:

  • gse – an object that contains information for gse element

  • TIME – the current simulation time

  • DFLAG – should be ignored

  • IFLAG – indicates why the routine is being called

  • NO – the number of outputs in the gse

  • Y – an array that contains the return values

Utility Functions#

Also, in kinetics_c_utils.h there is the declaration for utility subroutines c_sysary, c_sysfnc, c_errmes. From your subroutines, you can call c_sysary and c_sysfnc for access in kinetics-solver data. The function c_errmes provides the facility for outputting error messages from the user-defined subroutines.

void c_errmes(int errflg, const char *mesage, int id, const char *endflg)#

this function provides the facility for outputting error messages from user-written subroutines

Parameters:

  • errflg – a logical variable that is true if error occured

  • mesage – a character array with max size 1024, that errmes will document if errflg is true

  • id – the id of the statement that is invoking the user-written subroutine

  • endflg – a character array that stops execution when it equals to “STOP”

void Fiesub(const struct sAnsaField *fie, double time, double *disp, double *velo, int dflag, int iflag, double *field, double *dfddis, double *dfdvel)#

computes the force and torque components for a FIELD statement.

Parameters:

  • fie – an object that contains informations for field statement

  • time – the current simulation time

  • disp – an array that contains the translational and rotational displacements of the I-marker with respect to the J-marker expressed in the coordinate system of J-marker

  • velo – an array that contains the translational and rotational velocities of the I-marker with respect to the J-marker expressed in the coordinate system of J-marker

  • dflag – should be ignored (not supported yet)

  • iflag – indicates why the routine is being called

  • field – an array that contains the six return field components

  • dfddis – an array that contains the return derivatives of the six field components with respect to the six displacement values in disp array

  • dfdvel – an array that contains the return derivatives of the six field components with respect to the six velocity values in velo array

void Gfosub(const struct sAnsaGforce *gforce, double time, int dflag, int iflag, double *result)#

computes a set of six values for a GFORCE statement.

Parameters:

  • time – the current simulation time

  • dflag – should be ignored (not supported yet)

  • result – an array that contains the six return values
