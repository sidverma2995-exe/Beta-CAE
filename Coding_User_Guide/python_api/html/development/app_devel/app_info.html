Application Licensing and Distribution#

Introduction#

BETA CAE Systems has introduced a mechanism that allows Python script developers to license their applications. The licensing mechanism is based on the BETA LM system. The application developer will need to contact BETA through the ansa@beta-cae.com email. BETA will also provide a web portal where developers can advertise their applications and make them known to the BETA users’ community.

Licensing#

In order to license an application, a developer will need to contact BETA to create a BETA LM license feature name. This feature name can be used in the application’s code to manage the acquiring and the releasing of license credits. The two functions that handle this procedure are the ansa.session.AcquireFeature() and the ansa.session.ReleaseFeature()

In order to monitor the total and the available number of credits for a given license feature, the following BETA LM command may be used:

beta_lm -stat -h <the name of the license server>

A sample code is shown below.

def utilities_tools():

    ret_feature = session.AcquireFeature('UTILITIES_TOOLS')

    if ret_feature == -12:
        print('No credit is available')
        return 1

    if ret_feature == -10:
        print('This is an invalid feature')
        return 1

    _win()

    ret = session.ReleaseFeature('UTILITIES_TOOLS')

    print('Done')

The source code of a complete application can be found here. Utilities tool app

Note

The responsibility of the management of the license credits is upon the application’s developer. The developer needs to make sure that he acquires and releases the credits at the appropriate point in the code. If an application exits without releasing the credits, these credits will be locked for as long as the current ANSA session is alive.

Note

The application needs to be compiled (obfuscated) in order for the source code to be human unreadable. Please refer to Obfuscating an Application section.

Installation Packaging#

An Application can be packaged and distributed by two methods. The state of the art and the recommended method is the Apps and the ANSA Plugins Manager technology. The traditional Loading Scripts on ANSA startup procedure may also be used.
