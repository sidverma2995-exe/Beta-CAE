BCS Development Environment with Microsoft Visual Studio Code#

Introduction#

What is the BCS Development Environment#

The BCS Development Environment aims to provide an easy way to write, run and debug a Python script using an external IDE via the BETA CAE SUITE software. This is achieved by using a portable version of Visual Studio Code which comes preloaded with the necessary extensions for Python development along with an inhouse extension. The “bcs_dev_env” extension is designed to configure Visual Studio Code in order to communicate with the BETA CAE SUITE software. The BCS Development Environment is supported by BETA CAE SUITE v22.0.0 and above.

What is Visual Studio Code#

Visual Studio Code is a source code editor developed by Microsoft for Windows, Linux and macOS. It supports Python syntax, code completion and refactoring.

How to get Visual Studio Code#

Starting from version 23.0.0 a Visual Studio Code option has been added.

  • ANSA > Tools > Script > Visual Studio Code

  • META > Tools > Visual Studio Code

  • KOMVOS > Tools > Visual Studio Code

The first time you run it a wizard will be launched which will help you to install and configure Visual Studio Code. You have three options

Download from Microsoft#

You can use this option if you have Internet connection in order to install the latest Visual Studio Code and the necessary extensions. In the next page if you press the Download button a script will be executed. This script will download the latest version of Visual Studio Code, the bcs-dev-env-extension, it will install the necessary extensions such as the Python extension and will copy some required files. Finally it will create a package which will be used in the next step. This script can be found under ansa_vxx.xx.xx/scripts/BuildBcsDevEnv/DevEnvBuilder.py in case you want to run it manually in order to create the Visual Studio Code portable package.

From package#

You can use this option if you don’t have Internet connection. It is required to either run the ansa_vxx.xx.xx/scripts/BuildBcsDevEnv/DevEnvBuilder.py script externally and transfer the Visual Studio Code package or you can download from BETA CAE Systems website a preconfigured Visual Studio Codium package. It is advised to place this package under the BETA_CAE_SYSTEMS installation directory.

Use existing installation#

You can also use an existing Visual Studio Code installation in which you will install yourself the BCS Dev Env extension along with the necessary files. Information on how to do this can be found in the BETA CAE Systems website where you can download the extension. In the next step provide the path to the executable Code.exe (Windows) or code (Linux).

After installation#

If the installation completes successfully Visual Studio Code can run from the same button.

You can find the path to the Visual Studio Code executable in the settings, by going to Tools > Settings > Visual Studio Code. There, you can also click ‘Setup’ to run the installation wizard again. This way you can reconfigure or update to a newer version once it is available.

How to setup the BCS Debug Development Environment#

Starting from BETA CAE Suite v23.0.0 Visual Studio Code can be launched directly through ANSA, META and KOMVOS, using the Visual Studio Code button. In this case, the configuration will be completed automatically.

If you launch Visual Studio Code on your own, you must perform the setup manually. Follow the steps outlined below.

  1. Go to View > Command Palette… (Ctrl + Shift + P) and type BCS: Setup Development Environment.

  2. Selecting the Browse to a BCS installation entry a file manager will appear. Navigate to the location of ansa64.bat for Windows or ansa64.sh for Linux.

If you have previously configured a BCS installation, all BCS installations from the same directory will be listed. You can easily switch between the available BETA CAE Suite versions by running the setup command again or by pressing the BCS button in Visual Studio Code’s Status Bar.

How to run a Python script#

ANSA#

There are two scenarios:

  • In the first senario you first run ANSA and afterwards you launch Visual Studio Code via the Tools > Script > Visual Studio Code action.

  • In the second scenario you first run Visual Studio Code and afterwards you launch ANSA via the BCS: Launch ANSA Development Environment command or the blue run ANSA button at the top right toolbar of Visual Studio Code.

Open your script in Visual Studio Code, right-click within the editor or on the script’s file name in the Explorer panel, and select Load Python file in ANSA. You can now run the script using ANSA’s Script Manager or in any function that supports script execution.

META#

There are two scenarios:

  • In the first senario you first run META and afterwards you launch Visual Studio Code via the Tools > Visual Studio Code action.

  • In the second scenario you first run Visual Studio Code and afterwards you launch META via the BCS: Launch META Development Environment command or the red run META button at the top right toolbar of Visual Studio Code.

Open your script in Visual Studio Code, right-click within the editor or on the script’s file name in the Explorer panel, and select Run Python file in META.

KOMVOS#

There are two scenarios:

  • In the first senario you first run KOMVOS and afterwards you launch Visual Studio Code via the Tools > Visual Studio Code action.

  • In the second scenario you first run Visual Studio Code and afterwards you launch KOMVOS via the BCS: Launch KOMVOS Development Environment command or the green run KOMVOS button at the top right toolbar of Visual Studio Code.

Set the SDM_CONSOLE_PATH first in the settings of the extension. Open your script in Visual Studio Code and run it directly through KOMVOS.

How to debug a Python script#

ANSA#

  1. Open the script you want to debug in Visual Studio Code and add some breakpoints.

  2. Press Run > Start Debugging (F5) or select the BCS Remote Debug configuration in the Run panel. For the configuration to appear you must open a folder in the Explorer panel, not a single file. Wait until the Visual Studio code Status Bar turns orange.

  3. Now right click on the Python file you want to debug in the Explorer View and select Load Python File in ANSA.

  4. When you reach the breakpoint, you can return to Visual Studio Code and debug your script.

  5. While the debugger is attached in order to rerun a script in ANSA you can use Script Manager > Run. In META you can use the Run Python File in META command.

META#

  1. Open the script you want to debug in Visual Studio Code and add some breakpoints.

  2. Press Run > Start Debugging (F5) or select the BCS Remote Debug configuration in the Run panel. For the configuration to appear you must open a folder in the Explorer panel, not a single file. Wait until the Visual Studio code Status Bar turns orange.

  3. Now right click on the Python file you want to debug in the Explorer View and select Run Python File in META. Alternatively you can return to META and run the script execute your_script.py command.

  4. When you reach the breakpoint, you can return to Visual Studio Code and debug your script.

  5. While the debugger is attached in order to rerun a script in ANSA you can use Script Manager > Run. In META you can use the Run Python File in META command.

KOMVOS#

In KOMVOS you may want to set the SDM_CONSOLE_PATH first in the settings of the extension.

  1. Open the script you want to debug in Visual Studio Code and add some breakpoints.

  2. Press Run > Start Debugging (F5) or select the BCS Remote Debug configuration in the Run panel. For the configuration to appear you must open a folder in the Explorer panel, not a single file. Wait until the Visual Studio code Status Bar turns orange.

  3. Now return to KOMVOS and run your script.

  4. When you reach the breakpoint, you can return to Visual Studio Code and debug your script.

Remarks#

  • If you make changes to a script in Visual Studio Code and save it, the script will be immediately reloaded in ANSA once it has been loaded in the Script Manager. As a result, any global code will be executed right away.

  • When the debug process is started, stdout is forwarded to Visual Studio Code’s DEBUG CONSOLE. Until you restart the application, no print messages from the script will appear in ANSA/META Info.

  • If you stop the debugging process in Visual Studio Code, you won’t be able to resume it. ANSA, META or KOMVOS must be restarted.

  • When you launch Visual Studio Code, the extension will check for a newer BETA CAE Suite version and prompt you to run the BCS: Setup Development Environment command if one is available. This check can be turned off in the extension’s settings.
