NumPy#

NumPy in an extension of Python for scientific computing. It supports large multi-dimensional arrays and a large library of high level mathematical functions to operate on these arrays.

NumPy library v1.24.2 is embedded into the BETA installation. It uses MKL LP64 (32bit integers).

The user can access the library directly by importing it:

import numpy
numpy.show_config()

For more information on NumPy follow this link: NumPy

SciPy#

Scipy is a Python library for scientific computing. In contains modules for optimization, linear algebra, integration, interpolation, FFT, signal processing, ODE solvers, etc.

Version 1.10.1 is distributed within the BETA installation.

import scipy
from scipy import linalg
import numpy as np

def find_det():
    arr = np.array([[1, 2],
                    [3, 4]])

    det_res = linalg.det(arr)
    print(det_res)

OpenSSL#

OpenSSL is a toolkit library for TLS and SSL protocols. It is also a general purpose cryptography library.

The embedded version is 1.1.1t

H5py#

H5py is a pythonic interface to the HDF5 binary data format.

H5py version 3.8.0 is distributed within the BETA installation. Note that updating H5py to another version may fail if it is incompatible with the HDF library used by META.

randomcolor#

randomcolor, is a library that helps the programmer generate attractive random colors.

import randomcolor

def get_random_colors():
    random_color = randomcolor.RandomColor()
    colors =random_color.generate(format_='rgb Array', count=3)
    print(colors)

Installation/Management of non-embedded Libraries#

The installation of third party libraries can be a challenging task, as the libraries may have multiple dependencies and work only for specific python versions.

A package management system should be used, which helps us overcome these complexities, by automatically handling all the dependencies.

PIP#

pip is the standard package management system used to install and manage software packages written in Python.

It is the recommended system to use, as it comes pre-installed with BETA’s embedded Python interpreter and can be directly accessed to install and manage Python packages.

For example, if you want to install the pandas package, you can do the following.

Either from the command line:

cd /path/to/BETA_CAE_Systems/shared_v24.0.0/python

./python.sh -m pip install pandas

cd C:\path\to\BETA_CAE_Systems\shared_v24.0.0\python

.\python.bat -m pip install pandas

Or programmatically:

import pip

pip.main(['install', 'pandas'])

Warning

It is not recommended to install libraries programmatically, as Python will drop support for that in future versions and it will fail.

By default, pip will automatically download the specified package from Python Package Index, which is the standard repository of software for the Python programming language.

If your machine does not have an Internet connection, then you can manually download the distribution files and provide them to the pip command:

cd /path/to/BETA_CAE_Systems/shared_v24.0.0/python

./python.sh -m pip install sampleproject-1.0-py3-none-any.whl

cd C:\path\to\BETA_CAE_Systems\shared_v24.0.0\python

.\python.bat -m pip install sampleproject-1.0-py3-none-any.whl

In any scenario, pip will install the libraries in the BETA installation directory, making them directly accessible through an “import” statement.

import pandas

venv#

venv provides support for the creation of ‘virtual environments’ with their own site directories. Every environment is isolated with its own Python binary and its own independent installation of Python packages in its site directories.

Activating the needed virtual environment will allow the BETA Python interpreter to directly import packages from it. The user will not need to programmatically manage the importing.

First you have to create a new virtual environment:

cd /path/to/BETA_CAE_Systems/shared_v24.0.0/python

./python.sh -m venv /path/to/create/venv/my_test_venv --system-site-packages

cd C:\path\to\BETA_CAE_Systems\shared_v24.0.0\python

.\python.bat -m venv C:\path\to\create\venv\my_test_venv --system-site-packages

Inside the bin directory on Linux, or Scripts directory on Windows, you will find the scripts for all supported shells.

To activate the new environment, simply execute the following:

source /path/to/create/venv/my_test_venv/bin/activate

C:\path\to\create\venv\my_test_venv\Scripts\activate.bat

The activation will make the python of the environment active, so any pip installations that are performed will be for this virtual environment.

After activating the environment, all you have to do is open ANSA/META from within the venv’s directory.

The path of the virtual environment will be known to ANSA/META, together with the base environment, so all the installed packages can be directly imported.

import pandas

To deactivate the environment simply execute the following.

deactivate

C:\path\to\create\venv\my_test_venv\Scripts\deactivate.bat

CONDA Package and Environment Management System#

CONDA is a package management system that allows to easily install third party libraries that have a great amount of dependencies. The libraries that are part of the Conda system can then be imported into ANSA and META.

The following steps need to be made:

Download Conda#

Download and install Miniconda. Miniconda is a reduced version of Anaconda python distribution.

Conda Environment Installation and Set-up#

The Conda installation directory needs to be added to your system’s environment path.

export PATH="/home/username/miniconda3/bin":{$PATH}

setenv PATH $PATH:/home/username/miniconda3/bin

C:\Users\username\Miniconda3\Scripts

Conda’s environments are directory structures independent of each other that allow multiple versions of the same libraries to exist. ANSA and META are distributed with Python 3.11 embedded. Thus we need to create an environment with libraries that are Python 3.11 compatible.

conda create --name python311 python=3.11

Finally, the newly created environment needs to be activated so that the new installations will be for Python 3.11.

conda activate python311

Note

the activation of the virtual environment works only in bash

conda activate python311

Library Installation#

In order to install a library in an activated environment, simply type the following:

conda install library_name

Import into ANSA and META#

In order to import the Conda environment into your ANSA, or META script, append to the system path the Conda’s Python 3.11 site-packages. In order to retrieve the path of the site-packages, run the following with Conda’s python311 environment’s python alias:

python3 -c "import site; print(site.getsitepackages()[0])"

The path that is returned is used in ANSA and META to import the needed library.

import sys

CONDA_ENV = '/home/my_name/miniconda3/envs/python311/lib/python3.11/site-packages'
sys.path.append(CONDA_ENV)

import PIL

Note

Incompatibility issues might appear when trying to import libraries that are dependent on the C++ Standard Library, such as matplotlib, pandas, etc. In those cases conda should be avoided and the standard pip installation should be used instead.

Conda in a company’s Intranet#

In order to have the Python 3.11 compatible libraries inside a company’s Intranet, without access to the Internet, a Conda Channel needs to be set up. Users with individual miniconda installations will be able to install the needed library from the local Conda channel.

The general instruction to create and setup a custom channel are found here

A python script is provided, here, in order to setup a conda channel. The script runs in Python 3 and needs Internet connection in order to download the libraries required for the given python version.

After the channel is set up any user may access this channel and install a needed library. A user needs to have a local miniconda installation. The Conda needs to be set up to look up for the created channels and not the remote repositories on the Internet. To do that the .condarc located in the user’s home directory must look similar to this:

channels:
  - http://path/to_my/conda_channel/main/
  - http://path/to_my/conda_channel/free/

At this point the user can proceed per the instructions above and create a python 3.11 enviroment and install libraries.

Matplotlib Installation#

Special mention should be made of plotting libraries, such as Matplotlib, which is the widest used one.

The GUI libraries of Matplotlib, or any other plotting library, might lead to conflicts with BETA’s GUI libraries, so they should be treated carefully.

The recommended process to follow is to save the created plot on a file and then open it by using the system’s image viewer, or even create a BCWindow to display it.

The example below shows how to execute Matplotlib code in ANSA or META:

from ansa import guitk

import matplotlib
matplotlib.use('Agg')
from matplotlib import pyplot as plt

from PIL import Image

def my_plot():
    # Create the plot
    x_vals = [0.2, 0.7, 1.2, 2.7, 3.5]
    y_vals = [1.2, 1.8, 2.4, 3.9, 4.5]

    plt.suptitle('My Graph')
    ax = plt.subplot(111)
    ax.plot(x_vals, y_vals)

    # Save the plot to a file
    filename = '/path/to/image.png'
    plt.savefig(filename)

    # Use PIL to open the image in the default image viewer
    with Image.open(filename) as img:
        img.show()

    # Use the BCGUI library to show the image
    _show_image(filename)

def _show_image(filename):
    w = guitk.BCWindowCreate("Image Viewer", guitk.constants.BCOnExitDestroy)
    label = guitk.BCLabelCreate(w, '')
    guitk.BCLabelSetIconFileName(label, filename)
    guitk.BCShow(w)
