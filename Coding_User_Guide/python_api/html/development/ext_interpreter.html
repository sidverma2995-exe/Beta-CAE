External Interpreter#

Since version 24.1.0 an extra bash-script python-<app>.sh has been added to imitate a Python interpreter that can do ‘import ansa’, ‘import meta’ etc. This can be used in cases where a standard Python interpreter would be given.

The bash-scripts, which are only available for Linux machines, can be found in the following locations:

BETA_CAE_Systems/ansa_v24.1.3/python_ansa.sh
BETA_CAE_Systems/ansa_v24.1.3/python_komvos.sh
BETA_CAE_Systems/meta_post_v24.1.3/python_meta_post.sh
BETA_CAE_Systems/komvos_v24.1.3/python_komvos.sh
BETA_CAE_Systems/retomo_v24.1.3/python_retomo.sh

Command line usage#

You can use these scripts to easily run a Python script in nogui mode.

This is the command line usage of python_ansa.sh. Any parameter you give before the script file will be passed to ansa.sh as well.

$ /path_to/BETA_CAE_Systems/ansa_v24.1.0/python_ansa.sh
Usage: python_ansa.sh [option ... ] file.py

Here is an example test.py script:

import ansa
from ansa import base, constants

if __name__ == '__main__':
    base.CreateEntity(constants.NASTRAN, 'GRID')

You can run it like this:

$ /path_to/BETA_CAE_Systems/ansa_v24.1.0/python_ansa.sh test.py

Accessing command line arguments#

It is also possible to have access to all the command line arguments inside the script via sys.argv.

For example, running:

$ /path_to/BETA_CAE_Systems/ansa_v24.1.0/python_ansa.sh --undo no test.py

where test.py:

import sys

if __name__ == '__main__':
    print(sys.argv)

gives output:

['/path_to/BETA_CAE_Systems/ansa_v24.1.0/bin/ansa_linux_x86_64',
'--undo',
'no',
'-nogui',
'-exec',
'load_script:/home/username/test.py']

Shebang usage#

These scripts can be used as a shebang in a shell script. This way the script will be executed by using the Python interpreter embedded in the application and therefore we can use each application’s API.

Here is an example test.py script:

#!/path_to/BETA_CAE_Systems/ansa_v24.1.0/python_ansa.sh
import ansa
from ansa import base, constants

base.CreateEntity(constants.NASTRAN, 'GRID')

which can be run like this:

$ ./test.py
